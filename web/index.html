Communication Protocol

{
  "type": "start",          // start | single | stop
  "latched": true,          // latch on?
  "durations": [[1.5,1.0],[2.0,1.0],[2.5,1.0]]
}
// Client → HMI
{
  "status": "running|complete|stopped",
  "active_piston": 0,
  "direction": "extend|retract",
  "remaining_ms": 850,
  "cycle_count": 3,
  "latching": true
}

Flow

Command · HMI gathers durations, writes write_master_command.
Execute · Client reads the command, advances extend/retract stages (ms resolution).
State · Each step publishes write_slave_state with active piston, remaining time, latch, message.
Latch · If latched=true, waits 0.5 s after completion and restarts.
